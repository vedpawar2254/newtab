    - initialize() loads columns from settings (or creates default 3: To Do, In Progress, Done) and loads all tasks from db.tasks
    - addTask(title) creates a TaskRecord with columnId set to the first column's id and order at end
    - updateTask(id, updates) updates title and persists to db.tasks
    - deleteTask(id) removes task from store and db.tasks
    - completeTask(id) sets task.columnId to the last column's id (Done)
    - uncompleteTask(id) sets task.columnId to the first column's id (To Do)
    - reorderTask(id, newColumnId, newOrder) moves task to new column at new position, re-indexes affected tasks
    - addColumn(title) creates a new KanbanColumn, persists to settings
    - renameColumn(id, title) updates column title, persists to settings
    - deleteColumn(id) moves all tasks in that column to first remaining column, prevents deleting last column
    - reorderColumn(id, newOrder) reorders columns, persists to settings
    - All mutations persist immediately to Dexie (no debounce needed for tasks - they are small discrete writes)
  </behavior>

    1. Create lib/stores/task-store.ts following the notes-store pattern:

    ```typescript
    import { create } from 'zustand';
    import { db } from '../storage/db';
    import type { TaskRecord, KanbanColumn, KanbanState } from '../storage/types';

    interface TaskState {
      tasks: TaskRecord[];
      columns: KanbanColumn[];
      initialized: boolean;
      activeView: 'editor' | 'kanban';

      initialize: () => Promise<void>;
      addTask: (title: string) => Promise<TaskRecord>;
      updateTask: (id: string, updates: Partial<Pick<TaskRecord, 'title'>>) => Promise<void>;
      deleteTask: (id: string) => Promise<void>;
      completeTask: (id: string) => Promise<void>;
      uncompleteTask: (id: string) => Promise<void>;
      reorderTask: (id: string, newColumnId: string, newOrder: number) => Promise<void>;
      addColumn: (title: string) => Promise<KanbanColumn>;
      renameColumn: (id: string, title: string) => Promise<void>;
      deleteColumn: (id: string) => Promise<boolean>;
      reorderColumn: (id: string, newOrder: number) => Promise<void>;
      setActiveView: (view: 'editor' | 'kanban') => void;
      getDoneColumnId: () => string | undefined;
      getFirstColumnId: () => string | undefined;
    }
    ```

    Key implementation details:
    - Column IDs use `col-${crypto.randomUUID()}` prefix to avoid collision with task IDs
    - Task IDs use `task-${crypto.randomUUID()}` prefix
    - Default columns created in initialize():
      ```
      { id: `col-${crypto.randomUUID()}`, title: 'To Do', order: 0 }
      { id: `col-${crypto.randomUUID()}`, title: 'In Progress', order: 1 }
      { id: `col-${crypto.randomUUID()}`, title: 'Done', order: 2 }
      ```
    - Columns stored in settings table with key `kanban-columns` as KanbanState
    - completeTask: finds last column by max order, sets task.columnId to it
    - uncompleteTask: finds first column by min order, sets task.columnId to it
    - deleteColumn: if columns.length <= 1, return false (prevent). Otherwise move tasks to first remaining column, remove column, persist both
    - reorderTask: update the moved task's columnId and order, then re-index all tasks in affected columns (set order = array index)
    - persistColumns helper: saves columns to db.settings.put({ key: 'kanban-columns', value: { columns, updatedAt: Date.now() } })
