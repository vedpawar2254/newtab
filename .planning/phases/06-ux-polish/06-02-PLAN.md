1. Create `hooks/useKeyboardNav.ts`:
   - Export `useKeyboardNav()` hook that manages region-based Tab navigation
   - Define regions as data attributes: `data-region="sidebar"`, `data-region="editor"`, `data-region="panels"`
   - On Tab press (when not inside an input/textarea/contenteditable that handles Tab normally):
     - Find current active region by checking `document.activeElement` ancestry for `data-region`
     - Move focus to next region in order: sidebar -> editor -> panels -> sidebar
     - Shift+Tab reverses the order
     - When entering a region, focus the first focusable element within it (or the region container if it has tabIndex)
   - This hook should be called once in AppShell
   - Export the hook and a constant `REGION_ORDER = ['sidebar', 'editor', 'panels']`

2. Update `components/sidebar/PageTreeItem.tsx`:
   - Add `tabIndex={0}` to each tree item element
   - Add `role="treeitem"` for accessibility
   - Add `aria-expanded` attribute based on whether the item has children and is expanded/collapsed
   - Add `onKeyDown` handler:
     - `ArrowDown`: focus next visible tree item sibling (next sibling in DOM, or first child if expanded). Use `element.nextElementSibling` or parent traversal.
     - `ArrowUp`: focus previous visible tree item (previous sibling's last visible descendant, or parent)
     - `ArrowRight`: if has children and collapsed, expand (call `toggleCollapsed(id)`). If already expanded, focus first child.
     - `ArrowLeft`: if expanded, collapse. If collapsed or leaf, focus parent tree item.
     - `Enter`: activate the page (call `setActiveNote(id)`)
     - `Home`: focus first tree item in the entire tree
     - `End`: focus last visible tree item in the entire tree
   - Use `data-tree-item-id={id}` attribute so items can be found via querySelector

3. Update `components/sidebar/PageTree.tsx`:
   - Add `role="tree"` to the tree container element
   - Add `aria-label="Page tree"` for accessibility

4. Update `components/layout/Sidebar.tsx`:
   - Add `data-region="sidebar"` to the sidebar container element
   - Ensure the sidebar container or first focusable child can receive focus when Tab navigates to the sidebar region
