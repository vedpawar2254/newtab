# Phase 2 — UI Design Contract

> Visual and interaction contract for the Core Editor phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | radix-ui (primitives, inherited from Phase 1) |
| Icon library | lucide-react |
| Font | JetBrains Mono (with `ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, monospace` fallback) |

**Continuity:** Inherits all Phase 1 design system tokens, spacing, color palette, and animation conventions. This spec defines only editor-specific additions.

---

## Spacing Scale

Inherited from Phase 1 (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, toolbar button padding, inline code padding |
| sm | 8px | Toolbar button gaps, slash menu item padding-y, block vertical gap, table cell padding-y |
| md | 16px | Editor body padding, slash menu padding, code block padding, table cell padding-x |
| lg | 24px | Editor top padding below title, section breaks |
| xl | 32px | Editor horizontal padding |
| 2xl | 48px | Editor top margin (below app shell header region) |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- Drag handle hit area: 24px x 24px (centered on a 16px grip icon, provides comfortable grab target)
- Bubble toolbar button size: 32px x 32px (on-scale, provides comfortable touch/click target)

---

## Typography

Consolidated to 4 unique sizes with 2 weights:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 600 (semibold) | 1.2 | Note title field (H1 equivalent), Editor H1 blocks |
| Heading | 20px | 600 (semibold) | 1.3 | Editor H2 blocks, Editor H3 blocks |
| Body | 14px | 400 (regular) | 1.7 | Editor paragraph text, list items, task items, inline code, code block content, slash menu item labels, table cell text |
| Label | 12px | 400 (regular) | 1.4 | Toolbar labels, slash menu category headers, code block language label, link embed description, link embed URL, slash menu item descriptions |

**Notes:**
- Editor body uses 1.7 line-height (elevated from the 1.5 shell default) for comfortable reading in long-form content
- Only two font weights loaded: 400 and 600 (same as Phase 1)
- Heading letter-spacing: -0.01em (inherited from Phase 1)
- Note title field uses placeholder weight 400 for "Untitled" ghost text, switches to 600 on input
- H3 uses the same 20px Heading size as H2 but at weight 400 to create visual distinction
- Inline code and code blocks use Body size (14px) in monospace font for consistency
- Code block line-height: 1.6 (slightly tighter than body prose for code readability)

### Accent Reserved For (Phase 2 additions)

Accent color `#5B9BD5` is used exclusively on these editor elements:
1. Active formatting button state in bubble toolbar (icon color when bold/italic/etc. is active)
2. Links in editor content (text color)
3. Slash menu selected/highlighted item background: `rgba(91, 155, 213, 0.12)`
4. Task list checked checkbox fill
5. Cursor caret color (inherited from browser, but matching accent when styled)

Accent is NOT used for: code block backgrounds, table borders, slash menu backgrounds, block text, or decorative purposes.

| Property | Value |
|----------|-------|
| Trigger | Appears when text is selected (non-empty selection) |
| Position | Above selection, centered horizontally, 8px gap from text |
| Background | `#252525` |
| Border | 1px solid `rgba(255, 255, 255, 0.12)` |
| Border-radius | 8px |
| Padding | 4px |
| Shadow | `0 4px 12px rgba(0, 0, 0, 0.40)` (exception to Phase 1 flat-surface rule -- floating elements need depth) |

| Icon size | 16px |
| Divider | 1px solid `rgba(255, 255, 255, 0.08)`, 16px height, 4px horizontal margin |
| Entry animation | Fade in + scale from 0.95, 150ms ease-out |
| Exit animation | Fade out, 100ms ease-in |
| Hide delay | 100ms delay before hiding (prevents flicker during selection adjustment) |

**Button order (left to right):**
Bold | Italic | Underline | Strikethrough | `|` | Code | Link | `|` | Heading dropdown (H1/H2/H3)

**Accessible labels for icon-only toolbar buttons:**

| Button | `aria-label` |
|--------|-------------|
| Bold | `"Toggle bold"` |
| Italic | `"Toggle italic"` |
| Underline | `"Toggle underline"` |
| Strikethrough | `"Toggle strikethrough"` |
| Code | `"Toggle inline code"` |
| Link | `"Insert link"` |
| Heading | `"Change heading level"` |

**Heading dropdown:** Clicking the heading button opens a small dropdown below it with three options (H1, H2, H3). Each shows the label and size visually. Selecting one converts the current block.

**Exclusion zones:** Bubble toolbar does NOT appear when cursor is inside:
- Code blocks
- Image blocks
- Table cells (table has its own controls)

## Interaction States

### Editor Focus

| State | Behavior |
|-------|----------|
| Page load | Editor is NOT auto-focused (preserves Chrome address bar focus for new tab users) |
| First click | Click anywhere in editor body to focus; cursor placed at click position |
| Title to body | Press Enter in title field to move focus to first line of editor body |
| Body to title | Press Up Arrow when cursor is at the very start of first block |

### Block Hover

| State | Behavior |
|-------|----------|
| Mouse enters block | Drag handle fades in (100ms) on the left |
| Mouse leaves block | Drag handle fades out (100ms) |
| Drag handle hover | Icon color brightens from 0.25 to 0.50 opacity |
| Drag initiated | Block gets subtle outline, ghost follows cursor |
| Drop | Block repositioned, 200ms ease transition for surrounding blocks |

### Bubble Toolbar States

| State | Behavior |
|-------|----------|
| Text selected | Toolbar appears above selection (150ms fade + scale) |
| Selection empty | Toolbar hides (100ms delay, then 100ms fade) |
| Button hover | Background `#2F2F2F` |
| Button active (formatting on) | Icon `#5B9BD5`, bg `rgba(91, 155, 213, 0.12)` |
| Heading dropdown open | Small dropdown below heading button with H1/H2/H3 options |

### Slash Menu States

| State | Behavior |
|-------|----------|
| `/` typed | Menu appears below cursor (120ms slide + fade) |
| Typing after `/` | Items filter in real time, selected item resets to first match |
| No matches | "No match found -- press Esc to close" shown |
| Arrow navigation | Selected item highlighted, menu scrolls to keep selection visible |
| Enter on item | Block inserted, slash text removed, menu closes |
| Escape | Menu closes, `/` text remains |
| Click outside | Menu closes, `/` text remains |
| Backspace deleting `/` | Menu closes |

## Animation Contract

Inherits Phase 1 animation conventions. All animations respect `prefers-reduced-motion: reduce`.

| Animation | Duration | Easing | Property |
|-----------|----------|--------|----------|
| Bubble toolbar enter | 150ms | ease-out | opacity, transform (scale 0.95 to 1) |
| Bubble toolbar exit | 100ms | ease-in | opacity |
| Slash menu enter | 120ms | ease-out | opacity, transform (translateY 4px to 0) |

| Drag handle fade-out | 100ms | ease | opacity |

| Code copy feedback | Instant appear, 2000ms visible, 200ms fade-out | ease | opacity |
