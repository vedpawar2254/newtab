Create `components/editor/TableControls.tsx`:
- This is a CSS-only solution: use Tiptap's built-in table rendering, overlay + buttons with CSS positioning. NOT a custom NodeView (tables are complex in ProseMirror; wrapping them in a custom NodeView is risky).
- Instead, create a React component that renders alongside the editor and adds table controls via CSS and a thin wrapper.

**Approach: CSS + global editor styles for table controls**

Since Tiptap Table renders standard `<table>` elements in the ProseMirror DOM, add CSS-based + button overlays. The approach:

1. Add CSS rules in `entrypoints/newtab/style.css` for table hover controls:
```css
.ProseMirror .tableWrapper {
  position: relative;
  margin: 8px 0;
}

.ProseMirror .tableWrapper::after {
  /* Add row button area */
  content: '+';
  position: absolute;
  bottom: -12px;
  left: 50%;
  transform: translateX(-50%);
  width: 20px;
  height: 20px;
  display: flex;
  align-items: center;
  justify-content: center;
  background: #2F2F2F;
  border: 1px solid rgba(255, 255, 255, 0.12);
  border-radius: 50%;
  color: rgba(255, 255, 255, 0.50);
  font-size: 12px;
  cursor: pointer;
  opacity: 0;
  transition: opacity 100ms ease;
}

.ProseMirror .tableWrapper:hover::after {
  opacity: 1;
}
```

However, CSS `::after` can't trigger editor commands. So instead:

2. Create `TableControls` as a React component that:
   - Accepts `editor` prop
   - Uses a MutationObserver or editor event to detect when cursor is in a table
   - When editor selection is inside a table: render two floating buttons positioned relative to the table DOM element
   - **Add Row button:** positioned centered below the table bottom edge
     - 20px diameter circle, bg #2F2F2F, border 1px solid rgba(255,255,255,0.12), `Plus` icon 12px
     - Hover: bg #3A3A3A
     - `aria-label="Add row"`
     - On click: `editor.chain().focus().addRowAfter().run()`
   - **Add Column button:** positioned centered right of the table right edge
     - Same styling as Add Row
     - `aria-label="Add column"`
     - On click: `editor.chain().focus().addColumnAfter().run()`
   - Only visible when hovering the table area (use CSS group hover or mouse enter/leave state)
   - Position using `getBoundingClientRect()` of the table element found via `editor.view.domAtPos()` or by querying `.ProseMirror table`

3. Use `useEffect` with `editor.on('selectionUpdate')` to detect when cursor enters/leaves a table. Get the DOM node of the table to position buttons.

4. Add CSS for table cell selection highlight: `.ProseMirror td.selectedCell, .ProseMirror th.selectedCell { background: rgba(91, 155, 213, 0.10) }`

5. Add CSS for column resize handle: `.ProseMirror .column-resize-handle { background: rgba(91, 155, 213, 0.40); width: 3px; }`

Wire into Editor.tsx:
- Import TableControls
- Render `<TableControls editor={editor} />` inside the editor container

Update table CSS in style.css:
- Ensure table header row has bg #252525, font-weight 600
- Min column width: 80px via `.ProseMirror td, .ProseMirror th { min-width: 80px }`
- Table outer border-radius: 4px on the `.tableWrapper` element
