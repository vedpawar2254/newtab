Create `lib/editor/schema-migrations.ts`:
- Export `CURRENT_SCHEMA_VERSION = 1`
- Export interface `StoredDocument { schemaVersion: number; doc: any }`
- Export `serializeContent(editor: Editor): string` -- calls `editor.getJSON()`, wraps in `{ schemaVersion: CURRENT_SCHEMA_VERSION, doc: json }`, returns `JSON.stringify()`
- Export `deserializeContent(raw: string): any` -- parses JSON, checks schemaVersion, runs migration chain if needed, returns the `doc` field
- Handle empty string input (new note) by returning null
- Handle plain JSON objects (legacy v0 content without wrapper) by treating as v1

Create `lib/hooks/use-editor-autosave.ts`:
- Export `useEditorAutosave(editor: Editor | null, noteId: string | null)` hook
- Inside useEffect: if editor and noteId exist, listen to editor `update` event
- On update: call `serializeContent(editor)` to get JSON string, then call `useNotesStore.getState().updateNote()` passing the full NoteRecord with updated content string and title
- The 300ms debounce is already in StorageService, but add a local 300ms debounce wrapper here too to avoid excessive serialization. Use a simple setTimeout/clearTimeout pattern (NOT importing the debounce util -- keep it self-contained)
- On cleanup: remove listener, flush pending (call `handler.flush()` equivalent)
- Do NOT auto-focus the editor (anti-pattern from research: steals Chrome address bar focus)

Create `lib/hooks/use-note-loader.ts`:
- Export `useNoteLoader(editor: Editor | null, noteId: string | null)` hook
- When noteId changes: load note via `useNotesStore.getState().getNote(noteId)`
- If note found: call `deserializeContent(note.content)` and `editor.commands.setContent(result ?? '')` to swap content without recreating editor
- If note not found or noteId is null: `editor.commands.setContent('')`
- Return `{ note, isLoading }` state for title field to use
