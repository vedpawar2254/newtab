Create `entrypoints/background.ts`:
- WXT background service worker entrypoint: `export default defineBackground(() => { ... })`
- Listen for messages via `chrome.runtime.onMessage.addListener`
- Handle message type `'fetch-metadata'` with `{ url: string }` payload
- Fetch the URL using `fetch(url)` with a 5-second timeout (`AbortController`)
- Parse the HTML response as text
- Extract OG metadata using regex (no DOM parser in service worker):
  - `og:title` or `<title>` tag
  - `og:description` or `<meta name="description">`
  - `og:image`
  - Favicon: `<link rel="icon"` href, or fall back to `${origin}/favicon.ico`
- Return `{ title, description, image, favicon, url }` via `sendResponse`
- On any error (fetch fail, timeout, parse error): return `{ error: true, url }` so the client can fall back to plain URL
- Important: return `true` from the listener to indicate async response

  FileHandler.configure({
    allowedMimeTypes: ['image/png', 'image/jpeg', 'image/gif', 'image/webp'],
    onPaste: (editor, files) => {
      files.forEach(file => {
        const reader = new FileReader();
        reader.onload = () => {
          editor.chain().focus().setImage({ src: reader.result as string }).run();
        };
        reader.readAsDataURL(file);
      });
    },
    onDrop: (editor, files, pos) => {
